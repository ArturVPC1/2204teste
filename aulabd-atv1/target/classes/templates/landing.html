01

O que o sistema oferece

🗄️

Persistência robusta

Dados salvos no PostgreSQL com UUID gerado direto pelo banco. O schema sobe junto com a aplicação, sem configuração manual.

📋

Formulário inteligente

Thymeleaf vincula os campos do HTML direto ao objeto Java. O POST chega, o DAO insere, o usuário vê a confirmação.

⚙️

Arquitetura limpa

Controller, Service, DAO — cada camada faz uma coisa. Fácil de entender, fácil de mexer.

02

Como funciona

Requisição HTTP

O navegador faz um GET /aluno. O PaginaController recebe e injeta um objeto Aluno vazio no modelo.

Renderização

Thymeleaf processa o template formaluno.html e gera o HTML final com os campos vinculados ao objeto.

Submissão

O POST /aluno desserializa o formulário para um Aluno. O Service delega ao DAO que executa o INSERT via JdbcTemplate.

Confirmação

O controller redireciona para a view sucesso.html, confirmando ao usuário que o aluno foi cadastrado com sucesso.
